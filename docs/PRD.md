# PRD — 오늘의 기억 (데일리 두뇌 챌린지)

> 제품 요구사항 정의서. "무엇을, 왜" 만드는지의 기준 문서.
> 기능 추가·변경 여부를 판단할 때 이 문서를 기준으로 결정한다.

## 1. 한 줄 정의
같은 네트워크(부서) 안에서 두뇌 게임을 즐기고 점수 순위를 겨루는 **게임 허브**.
"한 끗 차이로 졌다"는 분함으로 재도전과 광고 시청을 유도한다.
- **게임① 오늘의 기억**: 빛나는 칸을 잠깐 보여주고 똑같이 누르는 격자 기억 게임
- **게임② 사자성어 도전**: 뜻을 보고 알맞은 사자성어를 4지선다로 고르는 게임
- 진입점은 **게임 선택 허브(`hub.html`)**, 순위는 게임별로 분리 저장

## 2. 목표 사용자 & 시나리오
- **주 사용자**: 같은 사무실/부서원 (같은 와이파이)
- **시나리오**: 호스트가 서버를 켜두고 주소를 공유 → 부서원이 접속해 하루 한 판 →
  순위표에서 서로 비교하며 경쟁

## 3. 핵심 심리 후크 (설계 의도)
| 후크 | 구현 |
|---|---|
| 두뇌회전 | 격자 패턴 순간 기억 |
| 자존심 | 기억력 나이 · 백분위 · 부서 리그(다이아/골드/실버/브론즈) |
| 경쟁심 | 같은 네트워크 실시간 누적 순위표 |
| 광고 유도 | "광고 보고 마지막 라운드 재도전", "광고 보고 정밀 분석" |

## 4. 게임 규칙 (현재 구현 기준)
### 게임① 오늘의 기억 (memory)
- **일반 모드** (scoreMul ×1): 5라운드, 격자 3×3→5×5로 확대, 타일 3→6개,
  노출 1500ms→1000ms로 단축
- **하드 모드** (scoreMul ×5): 단판, 7×7 격자에 11칸 번쩍 (2600ms 노출)
- **점수**: 정확도 + 속도 기반, 모드별 배수 적용

### 게임② 사자성어 도전 (idiom)
- 뜻(설명)을 보여주고 4지선다 중 알맞은 사자성어 선택
- **10문제 · 문제당 5초** 제한, 오답/시간초과는 0점이나 게임은 끝까지 진행
- **점수**: 정답당 100점 + 남은 시간 비례 속도 보너스(최대 100), 만점 약 2000
- 문제·오답 보기는 매판 랜덤 (내장 사자성어 55개에서 추출)

### 공통
- **닉네임**: 접속 시 입력 → 브라우저(localStorage `memgame_name`)에 저장, 두 게임 공유
- **순위 저장 단위**: 사람(id) + 게임(game) + 모드(mode)별 **최고 점수 1개만** 유지, 플레이 횟수 누적
- `game` 필드 없는 기존 기록은 자동 `memory`로 취급 (하위호환)

## 5. 저장/공유 모델 (중요)
| 접속 방식 | 순위 저장 | 비고 |
|---|---|---|
| 파일 직접 열기 (`file://`) | ❌ 브라우저 로컬만 | "전 세계 기준" 가짜 순위 표시 |
| 같은 네트워크 `http://호스트IP:8000` | ✅ 호스트 `scores.json` | 진짜 공동 순위 |
| GitHub Pages (`https://…github.io`) | ❌ 게임만 됨 | 파이썬 서버 실행 불가 |

→ **진짜 순위 공유는 호스트 PC + 서버가 켜져 있을 때만** 가능. 이건 구조적 제약.

## 6. 비목표 (지금은 안 함)
- 인터넷 전역 순위 (클라우드 DB 필요 — 별도 결정 사항, PLAN 참고)
- 로그인/회원 시스템 (닉네임만 사용)
- 실제 광고 SDK 연동 (현재는 후크 UI만)

## 7. 열린 결정 사항
- **PC 상시 가동 부담** → 클라우드(Supabase/Firebase) 전환 여부 (사용자 미결)
- 본인 순위 줄에 `(나)` 표시 추가 여부
