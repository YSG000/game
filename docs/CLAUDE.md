## 소통 원칙 (사용자 피드백에서 확정된 것)
1. **한국어로 먼저 묻고 진행한다.** 파일 생성·명령 실행 전에 말로 먼저 설명·확인.
2. **없으면 "없다"고 말로 답한다.** 명령어(코드)만 던지지 않는다.
3. **한 사이클이 끝나면 결과를 반드시 알려준다.** 되든 안 되든, 실패면 "실패했다"고 명확히.
4. **멍하니 있지 않는다.** 진행 중이면 진행 중이라고, 막혔으면 막혔다고 즉시 말한다.

## 작업 원칙
5. **기준은 사용자가 만든 원본.** `memory_daily_prototype.html`이 최종본이자 기준.
   맘대로 갈아엎지 말고, 있는 것을 유지하며 개선/추가한다.
6. **변경은 사용자가 요청한 것만.** 임의로 게임 형태를 바꾸지 않는다.
7. **파일 직접 수정 시 원본 보존.** 실험/배포본은 복사본으로 만든다 (예: `index.html`).
